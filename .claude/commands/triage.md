このIssueのトリアージを実施してください。

## 手順

1. Issue のタイトルと本文を分析する
2. 以下の「種類ラベル」から最も適切なものを **1〜2個** 選んでアサインする
3. 以下の「優先度ラベル」から最も適切なものを **1つ** 選んでアサインする
4. トリアージ結果を下記フォーマットでコメントとして投稿する

## 利用可能なラベル

### 種類ラベル（1〜2個選択）

| ラベル名 | 用途 |
|---------|------|
| `07:機能追加` | 新規機能の要望 |
| `06:機能変更` | 既存機能の改修要望 |
| `05:不具合修正` | バグ・不具合の報告 |
| `04:リファクタリング` | コード品質・構造の改善 |
| `03:デザイン` | UI/UX・見た目の調整 |
| `02:保留` | 解決の目途が立たない・情報不足 |
| `01:その他` | 上記に当てはまらない |

### 優先度ラベル（1つ選択）

| ラベル名 | 基準 |
|---------|------|
| `10:解決優先度(高)` | サービス障害・セキュリティ問題など緊急修正必須 |
| `09:解決優先度(中)` | 影響範囲が広い・重要機能に関わる問題 |
| `08:解決優先度(低)` | 軽微な問題・要望レベル |

## コメントフォーマット

以下のフォーマットでコメントを投稿してください：

```
## トリアージ結果

**種類**: [選択したラベル名]
**優先度**: [選択した優先度ラベル名]

**分析**:
[Issueの内容を1〜3文で要約し、分類の根拠を説明]

**対応方針**:
[推奨する次のアクション（例：再現手順の確認、仕様の検討、実装担当者のアサイン等）]
```
